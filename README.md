# Multi-Model Ice-Ice Disease Classification in _Kappaphycus alvarezii_

A benchmark of **4 shallow machine learning classifiers** across **4 preprocessing configurations** (16 model-configuration combinations total) for binary classification of _Kappaphycus alvarezii_ seaweed as either **healthy** or showing **early signs of ice-ice disease**.

The project compares classifier performance under a 2x2 factorial ablation of two classical preprocessing techniques — **median filter** denoising and **CLAHE** contrast enhancement — using HSV color histograms as handcrafted features.

## Highlights

- Multi-model benchmark: SVM, Random Forest, KNN, Logistic Regression
- 2x2 factorial preprocessing ablation: baseline / median / CLAHE / both
- Domain-shift correction via combined stratified re-split
- One-Class SVM out-of-distribution gate for safe deployment
- Interactive Streamlit dashboard for live image classification

## Pipeline

```
Image
  -> Preprocessing  (Baseline | Median | CLAHE | Median + CLAHE)
  -> HSV Color Histogram  (96-dim, L1-normalized)
  -> StandardScaler + Classifier  (SVM | RF | KNN | LR)
  -> Prediction  (healthy | early_signs)
```

| Component               | Choice                                                        |
| ----------------------- | ------------------------------------------------------------- |
| Features                | HSV color histogram, 32 bins per channel (96-dim)             |
| Preprocessing (toggled) | 5x5 median filter, CLAHE (clipLimit=2.0, tileGridSize=8x8)    |
| Classifiers             | SVM (RBF), Random Forest, KNN, Logistic Regression — defaults |
| Pipeline                | StandardScaler + Classifier                                   |
| OOD detection           | One-Class SVM (gamma='scale', nu=0.05)                        |
| Evaluation              | Accuracy, Precision, Recall, F1-Score                         |
| Data split              | Stratified 80/20 with fixed random seed                       |

## Why default hyperparameters?

The study isolates the effect of preprocessing, not classifier tuning. All classifiers use default scikit-learn hyperparameters across all configurations, so any performance difference is attributable to the preprocessing step under test rather than per-classifier optimization.

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/ice-ice-disease-classification.git
cd ice-ice-disease-classification
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

Or using conda:

```bash
conda create -n ice_ice_dip python=3.11 -y
conda activate ice_ice_dip
pip install -r requirements.txt
```

### 3. Add the dataset

The dataset is not included in this repository due to size constraints. Place your dataset in the project root with the structure below:

```
dataset/
├── train/
│   ├── healthy/
│   └── early_signs/
└── test/
    ├── healthy/
    └── early_signs/
```

Note: this project pools train and test partitions into a single stratified random split. The original folder structure is preserved for compatibility but the actual train/test split is performed at the path level inside the notebook.

To request the dataset, contact the author.

### 4. Train all 16 classifiers + OOD gate

Open `svm_v7_clean.ipynb` in VS Code or Jupyter and click **Run All**. This will:

1. Load images from both train/ and test/ folders
2. Apply a stratified 80/20 random split to correct the domain shift in the original partitions
3. Extract HSV color histograms under all 4 preprocessing configurations
4. Train 4 classifiers x 4 configs = 16 binary classifiers
5. Train a One-Class SVM gate for out-of-distribution detection
6. Generate all visualizations (sample images, preprocessing comparison, confusion matrix grid, F1-score heatmap, multi-model overfitting diagnostic, PCA scatter plots)
7. Save all models + config to the `models/` folder

Total runtime: approximately 2-5 minutes depending on dataset size and hardware.

### 5. Launch the Streamlit dashboard

```bash
streamlit run app.py
```

Opens automatically at `http://localhost:8501`.

When you upload a seaweed image, the dashboard displays:

- KPI cards with benchmark summary
- Sidebar showing the best (classifier, configuration) combination
- Expandable training benchmark heatmap
- One-Class SVM out-of-distribution check (rejects non-seaweed images)
- Original image and preprocessing variants side-by-side
- Prediction matrix from all 16 combinations
- Detailed results table with confidence scores
- Consensus voting analysis
- Recommended final diagnosis (from the best combination identified during training)
- Per-classifier confidence comparison chart

## Methodology notes

### Domain-shift correction

Initial experiments using the provided train/test partition revealed catastrophic overfitting (>99% training accuracy, ~30% test accuracy) despite valid features and classifier setup. Diagnostic analysis using a shuffled re-split confirmed that the original train and test folders came from systematically different distributions (likely different acquisition sessions). The fix is implemented at the path level inside the data loading section: all image paths from both folders are pooled and split via `sklearn.model_selection.train_test_split` with `stratify=labels` and `random_state=42`. This ensures both partitions come from the same distribution, restoring valid evaluation.

### Out-of-distribution detection

The 16 binary classifiers are closed-set — they always assign one of the two trained classes to any input, even images that contain no seaweed. To prevent meaningless predictions in deployment, a separate One-Class SVM gate is trained on the seaweed training distribution. At inference time, the Streamlit dashboard runs this gate first and rejects out-of-distribution inputs (non-seaweed images) before they reach the binary classifiers.

### Why the high accuracy is genuine but in-distribution

Results are achieved on test data drawn from the same controlled acquisition setup as training (indoor, tripod-mounted Sony ZV-E10M2, fixed f/5.6 aperture, ISO 200-250, dark standardized background). This represents _in-distribution_ performance. Real-world deployment under field conditions (outdoor lighting, mobile phone cameras, water surface reflections) would likely show lower accuracy due to domain shift between research and deployment conditions. Future work should evaluate the pipeline on field-collected imagery to characterize deployment-time performance.
